アプリ要件定義書：MamaLink（仮）
1. プロジェクト概要
コンセプト： 「孤育て」をなくし、地域で助け合えるセーフティネットの構築。

ターゲット： 子育て中の母親（本人確認必須）。

主要価値： 信頼できる近所のママ同士で「あそぶ」「話す」「助け合う」を完結させる。
＠１
2. 機能要件
2.1 ユーザー管理・認証
会員登録： メールアドレス、SNS連携。

本人確認（必須）： eKYCによる公的身分証（免許証等）の提出。

女性であること、実在する人物であることを確認。

プロフィール設定：

子供の年齢（月齢）、人数、アレルギー、興味関心、居住エリア（市区町村まで表示）。

「助けられること」リスト（お使い、子守り、相談など）。

2.2 3つのモード（メイン機能）
① あそぼ（交流モード）
近所検索： 指定距離内のユーザーを一覧表示。

イベント作成： 「〇時〜〇〇公園で遊びませんか？」の投稿。

② おしえて（相談モード）
掲示板： カテゴリ別（離乳食、夜泣き、保活など）のQ&A。

匿名投稿： センシティブな悩み向けの匿名機能。

③ たすけて（助け合いモード：メイン）
依頼投稿： カテゴリ、日時、場所、お礼の有無、詳細内容の入力。

応募管理： 応募者の中から依頼者が1名を選択（承諾）。

ステータス管理： 「マッチング中」「活動開始（チェックイン）」「完了」。

2.3 安心・安全機能
通報・ブロック： 24時間365日の受付。

相互レビュー： 完了後の評価（5段階＋コメント）。

緊急連絡先登録： 助け合い開始時に双方に開示される設定。

AIメッセージフィルタリング： 連絡先の直接交換や不適切ワードの検知。

3. 「助け合い」のルールと決済
3.1 お礼（マネタイズ連携）
ボランティア： 「サンクスポイント」や「感謝メッセージ」の送付。

プチギフト： アプリ内で「デジタルギフト券（スタバ、コンビニ等）」を購入し、相手に贈呈（運営が手数料を徴収）。

有償依頼： 運営が定める一律料金（例：15分250円等）に基づき、アプリ内決済で支払い。

3.2 保険
活動付帯保険： 有料会員、または有償マッチング時に自動適用される賠償責任保険。

4. 収益化モデル（マネタイズ）
アプリ内広告： ターゲットに合わせたベビー・ライフスタイル商品の広告。

有料会員（サブスク）： * 広告非表示。

専門家（助産師等）へのチャット相談。

活動保険の自動付帯。

決済手数料： デジタルギフトや有償依頼の決済発生時の手数料（10〜20%）。

5. 画面一覧（主要画面）
スプラッシュ/ログイン画面

ホーム画面： 3モード（あそぼ・おしえて・たすけて）の切り替え。

「たすけて」投稿フォーム： 項目選択〜チェックリスト〜投稿。

チャット画面： マッチング後の個別連絡用。

プロフィール・マイページ： 信頼スコア、本人確認ステータスの表示。

管理画面（運営用）： ユーザー審査、通報対応。

6. 今後の検討事項（フェーズ2以降）
エリアアンバサダー制度： 地域ごとの信頼できるママリーダーによるオフライン面談。

企業クーポン配布： 地元の飲食店や施設との連携。
