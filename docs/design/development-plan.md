# 開発計画書 (Development Plan)

## 1. 開発ロードマップ

### Phase 1: MVP開発 (3ヶ月)
**目標**: コア機能である「助け合いマッチング」と「安全性」の検証。
- リリース対象: 特定エリア（パイロットエリア）。
- 主要機能: eKYC、3モード検索、1対1/グループマッチング、チャット、レビュー。

### Phase 2: 機能拡張 (2ヶ月)
**目標**: 利便性の向上とコミュニティの活性化。
- 有償依頼の決済連携（Stripe）、デジタルギフト機能。
- 専門家チャット相談。
- 活動付帯保険の自動適用。

### Phase 3: スケールアップ
**目標**: 対応エリアの拡大とエコシステム構築。
- エリアアンバサダー制度。
- 企業連携（クーポン配布）。

## 3. リスク管理

| リスク | カテゴリ | 影響度 | 対策 |
| :--- | :--- | :--- | :--- |
| マッチング時の事故・トラブル | 法的/安全 | 極大 | eKYCの徹底、保険導入、通報機能の即時対応。 |
| ユーザーのアンバランス | ビジネス | 大 | 初期サポーターの事前募集、紹介制度の導入。 |
| 個人情報の流出 | 技術 | 大 | Clerk/Supabaseによる強固なセキュリティ、データの最小限保持。 |

## 4. 運用方針
- **デプロイ**: Vercelによる自動デプロイ（Mainブランチマージ時）。
- **監視**: Vercel AnalyticsおよびSupabase Logsによる監視。
- **通報対応**: 初期は運営メンバーによる24時間通知監視。
